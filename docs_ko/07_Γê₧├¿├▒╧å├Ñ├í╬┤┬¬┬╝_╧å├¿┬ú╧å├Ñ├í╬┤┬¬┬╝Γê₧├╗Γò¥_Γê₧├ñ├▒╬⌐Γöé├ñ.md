# 07. 스토리/튜토리얼 설계

## 스토리 톤
- 정파/사파/상단/객잔 세력이 얽힌 무림 복구극
- 초반에는 거대한 세계관보다 `객잔 하나를 다시 살린다`는 작은 목표에 집중
- 최근 무협 인기작들이 자주 쓰는 `몰락 세력 재건`, `기연`, `문파 경쟁`, `성장형 인물군` 감성을 참고
- 다만 고유 명칭/사건/설정은 직접 차용하지 않고 오리지널 플롯으로 유지

## 초반 15분 플레이 목적
1. 객잔과 문파가 왜 망했는지 알게 한다.
2. 첫 헌터를 모집하고 사냥/귀환/회복 루프를 체험시킨다.
3. 광고는 `선택하면 편해지는 기능` 정도로만 소개한다.
4. 확률형 보상 표기는 별도 버튼에서 항상 볼 수 있게 한다.

## 튜토리얼 퀘스트 원칙
- 한 번에 3개 이하만 노출
- 주 목표 1개 + 보조 목표 1개 + 선택형 목표 1개 구조 권장
- 광고 관련 퀘스트는 반드시 `선택` 문구 포함
