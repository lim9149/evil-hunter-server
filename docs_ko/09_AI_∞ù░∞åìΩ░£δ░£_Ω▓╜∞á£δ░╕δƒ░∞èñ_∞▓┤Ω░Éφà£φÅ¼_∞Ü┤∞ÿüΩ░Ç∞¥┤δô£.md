# AI 연속개발 가이드 — 경제 밸런스 / 체감 템포 / 운영 흐름

## 이번 패키지에서 바로 이어서 볼 위치
- 서버 헌터 운영 규율: `server_source/core/hunter_operations.py`
- 서버 장기 경제 시뮬레이션: `server_source/core/economy.py`
- 서버 API: `server_source/routers/hunter.py`, `server_source/routers/world.py`
- 유니티 운영/혼잡 제어: `HunterBrain.cs`, `HunterTrafficCoordinator.cs`, `HunterSystemManager.cs`
- 유니티 입력흐름: `TownWorldInputFlowController.cs`

## 경제 밸런스 의도
- 전투력만 높다고 골드/경험치가 직선적으로 무한 상승하지 않도록 설계
- 사기(morale), 피로(fatigue), 휴식 규율(rest discipline)이 장시간 수익에 직접 반영되도록 설계
- 따라서 플레이어는 단순 육성만이 아니라 **운영 규율을 조정하는 재미**를 느끼게 됨

## 체감 템포 의도
- TownWorld에서는 사냥-귀환-회복-재출격이 한눈에 읽혀야 함
- 그래서 개별 루프를 짧게 유지하고, `uiStateLabel`, `huntLoopProgress`를 노출 가능 상태로 추가함
- 다음 AI는 HUD에 진행바/상태 태그를 붙이면 플레이 감각을 훨씬 빠르게 완성할 수 있음

## 혼잡도/경로 처리 의도
- 다수 헌터가 같은 포인트로 겹치면 조잡해 보이므로 lane offset과 facility slot reservation을 넣음
- 현재는 NavMesh 없는 상태를 가정한 경량 구현
- 유니티 사용 가능 시 NavMeshAgent 또는 로컬 회피로 고도화 권장

## 저작권/차별화 방향
- 다른 작품과 비슷해 보일 수 있는 단순 마을+자동사냥 구조를 피하기 위해,
  **무림 객잔 운영 / 규율 / 기강 / 회복 교리** 중심 구조를 코드에 반영함
- 다음 AI는 아래를 더 밀어주면 좋음
  1. 시설명을 무림 세계관 고유명으로 재정비
  2. 직업 계열명과 전직명 전면 차별화
  3. 이벤트/대사/UI 카피를 고유 톤으로 정리

## 2026-03-12 운영 경제 추가 기준
- 헌터 수익 = 총 드롭 골드 + 판매 가능한 재료 가치
- 문파 수수료 = 헌터별 사냥 순수익의 비율로 계산
- 권장 시작값: 초반 8%, 중반 12%, 후반 15%
- 수수료가 높아질수록 충성도/기분/이탈확률 또는 불만 대사에 영향
- 운영자 골드는 물약/무기/방어구 제작과 교육 투자에 재사용되어야 한다

## 운영자가 직접 만지는 축
1. 제작 큐 우선순위
2. 판매가/재고 유지선
3. 교육 투자
4. 전직 타이밍
5. 환골탈태 준비 자원
6. 수수료 정책
- `/hunters/{id}/operation-plan` 결과가 플레이 의도를 설명하는지
- `/world/economy/simulate`에서 피로 평균이 과도하지 않은지
- 유니티에서 `SelectNextHunter`, `FocusSelection`, `1/2/3 배속` 흐름이 바로 연결 가능한지
