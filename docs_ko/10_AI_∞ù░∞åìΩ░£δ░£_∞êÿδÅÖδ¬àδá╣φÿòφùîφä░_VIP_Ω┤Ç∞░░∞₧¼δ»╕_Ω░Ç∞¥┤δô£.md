# AI 연속개발 가이드 — 자율 헌터 AI 유지 + 직접 명령 공존 + 무협 차별화

## 가장 중요한 정정
이 프로젝트의 차별화 목표는 **헌터가 돌아다니고 사냥하는 매력 자체를 없애는 것**이 아니다.
정답은 반대로 아래다.

- 마을이 계속 살아 있어야 한다.
- 헌터는 자율적으로 돌아다니고, 쉬고, 훈련하고, 사냥하고, 귀환해야 한다.
- 플레이어는 필요할 때 특정 헌터에게 직접 명령을 내려 개입할 수 있어야 한다.
- 차별화는 이 생활 AI를 **무협풍으로 더 깊게** 만드는 데서 와야 한다.

## 이번 반영 내용
### 1) Unity 헌터 AI 복원 및 고도화
- `HunterBrain.cs`가 다시 **자율 루틴 판단**을 수행한다.
- 명령이 없을 때 헌터는 상태와 성향에 따라 다음 행동을 스스로 고른다.
  - 사냥
  - 훈련
  - 순찰 / 정세 파악
  - 휴식 / 식사 / 치료

### 2) 개별 성향 필드 추가
- `bravery`
- `sociability`
- `discipline`
- `preferredActivity`
- `directOrderActive`
- `lastCompletedRoutine`

### 3) 직접 명령은 유지
- `hunt / train / rest / eat / heal / patrol / return / hold`
- 단, 해석은 이제 아래와 같다.
  - **현재 자율 루틴을 잠시 끊고 수행**
  - **명령 완료 후 다시 자율 생활 루프로 복귀**

### 4) 혼잡/경로 분산 강화
- 사냥 lane offset 확장
- 시설 슬롯 원형 분산
- 순찰 지점 예약

### 5) 서버 연속개발 정보 강화
새 API:
- `GET /hunters/{hunter_id}/ai-profile`
- `POST /hunters/{hunter_id}/configure-ai`

저장 필드:
- `aiMode`
- `preferredActivity`
- `socialDrive`
- `disciplineDrive`
- `braveryDrive`

## 무협 차별화 기준
다음 AI는 절대 "마을 운영은 비슷하니 AI를 삭제하자"로 가면 안 된다.
차별화는 아래처럼 해야 한다.

### 좋은 차별화
- 객잔 중심의 생활 루프
- 문파/강호/풍문/의뢰판/순찰
- 헌터가 훈련당, 객잔, 약방, 게시판을 오간다
- 헌터마다 성격이 달라 행동이 다르다
- 전투 외에도 기강, 회복, 교류, 정보수집이 중요하다

### 피해야 할 차별화
- 헌터를 화면에서 안 보이게 만들기
- 전투를 숫자 처리만 하는 정적 메뉴형으로 바꾸기
- 마을이 멈춘 것처럼 보이게 만들기

## 다음 AI가 우선 구현할 것
## 2026-03-12 정정 - 직접 명령보다 운영 명령이 우선
- 플레이어가 자주 누르는 것은 전투 명령보다 운영 명령이어야 한다.
- 직접 명령은 예외 개입이고, 기본 루프는 `사냥터 지정 후 헌터 자동 수행`이다.
- 핵심 메뉴는 헌터 개별 조작보다 `문파 운영 패널`, `제작`, `판매`, `교육`, `전직`, `환골탈태` 쪽으로 무게를 둔다.
1. 실제 월드에 헌터별 말풍선/상태표시 연결
2. 훈련/사냥/순찰/휴식 애니메이션을 시각적으로 구분
3. 하단 카드에서 성향/현재 루틴/직접명령 상태 표시
4. 게시판/객잔/약방/훈련당/대장간을 실제 POI로 연결
5. 헌터별 관계/풍문/기강 시스템을 추가해 무협다움을 강화

## 이어받는 AI를 위한 한 문장 요약
이 게임의 기준선은 **"살아 있는 무림 객잔 마을 + 자율 헌터 AI + 직접 명령 개입 + 성향 기반 차별화"** 이다.
