# AI 연속개발 가이드 — 자율 헌터 AI / 실시간 마을 루프 / 건물 재배치

## 1. 사용자 의도 최종 정리
사용자가 원하는 것은 다음이다.

- 헌터가 마을에서 **AI로 스스로 돌아다니기**
- 헌터가 **실제로 사냥하러 이동하고 싸우고 돌아오기**
- 필요할 때만 유저가 **개별 헌터를 눌러 명령하기**
- 건물을 **마을 안에서 자유롭게 배치/이동하기**
- 마을이 커지면 카메라를 움직여 전체를 구경하기
- 헌터 상태에 따라 **혼잣말/말풍선**이 나오기
- 이 모든 것을 **실시간으로 보고 방치의 재미를 느끼기**

즉, 이 게임은 `버튼 눌러 숫자만 오르는 방치형`이 아니라,
**“살아 있는 무림 객잔과 사냥터를 관찰하는 운영형 타이쿤”** 이어야 한다.

## 2. 이번 코드 변경 요약
### 헌터 AI
- 자율 행동 선택 로직 추가
- 생활 상태 추가
  - 담소
  - 산책
  - 무공 수련
  - 제작/수리
  - 상점 구경
- 상태별 대사 필드 추가

### 유저 개입
- 클릭으로 근처 헌터 선택
- 선택 헌터에게 수동 명령 가능
- 명령은 기본 자율 루프를 덮어쓰는 방식

### 월드 구조
- 큰 마을을 위한 카메라 팬/드래그 준비
- 건물 재배치 시스템 프로토타입 추가
- 시설 위치를 런타임 기준으로 재해석하도록 수정

## 3. 현재 기준 추천 플레이 루프
1. 객잔/게시판에서 의뢰가 열린다.
2. 일부 헌터는 자율적으로 훈련/휴식/생활 행동을 한다.
3. 플레이어가 특정 헌터를 탭한다.
4. 사냥/무공 수련/제작/전직 등을 명령한다.
5. 헌터가 실제 월드에서 이동한다.
6. 전투/수련/제작 결과가 실시간으로 보인다.
7. 귀환 후 객잔/주점/의원에서 회복한다.
8. 다시 생활 AI 루프로 복귀한다.

## 4. 차별화 핵심 설계
### 유지해야 하는 공통 매력
- 헌터가 마을에서 돌아다니는 것
- 사냥 장면을 보는 것
- 개별 헌터를 키우는 맛

### 더 강하게 차별화할 부분
- 무림 객잔, 무공당, 의원, 대장간 중심의 생활 동선
- 전투 외에도 수련/제작/담소/정보 수집을 보는 재미
- 건물 재배치로 동선 효율이 달라지는 운영 깊이
- 상태 기반 대사와 개성 있는 생활 패턴

## 5. 다음 AI 구현 체크리스트
### UI
- 선택 헌터 하단 카드/플로팅 버튼
- 건물 배치 모드 UI
- 말풍선 프리팹
- 카메라 미니맵 또는 영역 포커스 버튼

### 애니메이션/씬
- 헌터 걷기/공격/회복/훈련 모션 연결
- 시설 도착 시 방향 전환 및 자리잡기
- 몬스터 존 타격/피격 타이밍 동기화

### 데이터
- 헌터 성격값 저장
- 건물 배치 좌표 서버 저장 여부 결정
- 생활 대사 테이블 추가

### 밸런스
- 동선이 너무 길면 재미가 죽으므로 마을 크기와 이동속도 균형 필요
- 자율행동 빈도와 수동 명령 빈도의 비율은 7:3 정도에서 시작 권장
- 담소/산책이 너무 잦으면 답답하므로 체감 템포 체크 필요
