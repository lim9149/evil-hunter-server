# DEV_DIRECTION_UI_LOCK_2026_03

## 목적
이 프로젝트의 앞으로의 개발 방향을 `세로형 픽셀 TownWorld 타이쿤 RPG`로 고정한다.
사용자가 공유한 스크린샷 감성은 참고하되, **에셋/배치/명칭/세부 수치의 직접 복제는 금지**한다.

## UI 방향 고정
- 화면 비율: **모바일 세로형(Portrait)** 우선
- 핵심 화면: **TownWorld 단일 Scene**
- 플레이 흐름: 마을 운영 + 자동 사냥 + 귀환 + 성장 + 오버레이 UI
- 전투는 별도 전환 화면보다 **월드 안에서 보이는 방식**을 우선
- HUD 원칙: `상단 자원`, `상단 퀵 아이콘`, `좌측 퀘스트`, `우측 플로팅 숏컷`, `하단 고정 메뉴`
- 하단 메뉴는 4~6개 수준의 큰 버튼으로 유지하고, 모바일 한 손 조작을 우선한다.
- 팝업/패널은 전체 씬을 갈아엎는 방식보다 **오버레이 패널**로 처리한다.

## 화면 구성 원칙
### 1) 상단 바
- 골드 / 프리미엄 재화 / 행동 재화 / 인구/슬롯 등 핵심 재화만 상시 노출
- 우측 상단에는 우편, 업적, 퀘스트, 설정, 이벤트 등 퀵 접근 아이콘 배치
- 아이콘은 많아도 읽기 가능하도록 정렬하되, 첫 화면은 과밀 복제를 피하고 단계적으로 오픈한다.

### 2) 중앙 월드
- 마을 중심 + 필드 연결 구조
- 헌터가 직접 이동하며 사냥/귀환/회복하는 흐름을 시각화
- 시설은 작은 픽셀 건물 단위로 배치하며, 클릭/탭 가능한 앵커를 둔다.
- 월드의 생동감은 `다수 캐릭터 이동`, `작은 이펙트`, `시설 상호작용`, `몬스터 스폰`으로 만든다.

### 3) 좌측 정보 영역
- 초반에는 튜토리얼/미션 추적 UI를 좌측에 세로 카드형으로 유지
- NPC 대사 / 가이드 / 보상 안내도 좌측 또는 하단 말풍선형 패널로 연결

### 4) 우측 플로팅 숏컷
- 자주 쓰는 모드 전환, 커뮤니티, 알림, 우선 행동 버튼은 우측 하단 원형/육각형 플로팅 버튼군으로 정리
- 단, 버튼 의미를 명확히 하기 전까지는 실제 기능보다 자리와 UX 구조를 먼저 고정

### 5) 하단 고정 메뉴
- 건설 / 던전 또는 필드 / 헌터 / 창고 / 상점 계열의 5버튼 구조를 기본 골격으로 삼는다.
- 명칭은 프로젝트 고유 세계관에 맞게 변경 가능하나, `직관적이고 반복 학습 가능한 구조`는 유지한다.

## 시스템 방향 고정
## 운영자 중심 추가 고정
- 플레이어는 헌터를 상시 직접 조종하지 않는다.
- 사냥의 미세 행동은 헌터 AI가 수행하고, 플레이어는 사냥터 지정과 운영 결정을 내린다.
- 하단 메뉴와 접이식 패널은 `문파`, `헌터`, `제작`, `상점`, `교육` 중심으로 읽혀야 한다.
- 전직/환골탈태/고급 규율은 단계 해금 구조로 설계해 초반 복잡도를 낮춘다.
- `건설`, `헌터 관리`, `자동 사냥`, `필드 진입`, `장비/성장`, `오프라인 보상`은 핵심 루프다.
- 헌터 최대 상주 수는 현재 문서 기준 20명을 유지한다.
- 헌터는 마을 안과 필드를 실제로 오가며, 상태가 나쁘면 시설로 복귀한다.
- 시설은 단순 수치 UI가 아니라 월드 오브젝트 + 패널 진입점이다.
- 광고는 오직 선택형 보상 광고만 허용하며 강제 인터럽트는 금지한다.

## 아트/연출 원칙
- 픽셀아트 + 저사양 친화 + 시인성 우선
- 작은 캐릭터도 클래스/등급/상태를 구분할 수 있어야 한다.
- 과한 파티클보다 색, 실루엣, 머리/어깨/장신구 포인트로 식별성을 만든다.
- 건물과 UI는 읽기 쉬운 대비를 유지하고 텍스트는 모바일에서 뭉개지지 않게 한다.

## 구현 우선순위
1. 세로형 TownWorld HUD 골격 완성
2. 하단 메뉴 / 상단 자원바 / 좌측 미션 / 우측 숏컷 고정
3. 건물 탭 진입과 헌터 자동 루프 연결
4. 월드 내 사냥 필드와 귀환 흐름 연결
5. 오버레이 패널(우편/공지/퀘스트/헌터정보/상점) 통일
6. 저사양 최적화와 터치 영역 보정

## 금지사항
- Evil Hunter Tycoon의 에셋, UI 배치, 명칭, 수치, 아이콘을 그대로 복제하지 않는다.
- 최근 무협 IP의 캐릭터/세계관/고유명사를 직접 차용하지 않는다.
- 전투 도중 강제 광고, 과도한 팝업, 다중 씬 난립 구조를 기본안으로 채택하지 않는다.
